# Design documents

Place **every** repository design artifact here:

- Threat models, phased roadmaps, implementation programs (checkbox plans), BPF/CI brainstorming outputs.

**Naming:** `YYYY-MM-DD-<topic>-design.md` or `YYYY-MM-DD-<topic>-implementation-plan.md`.

Do **not** store published design Markdown under `.github/design/` — that path stays gitignored to avoid stray copies; **`docs/design/`** is canonical.
