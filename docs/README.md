# Documentation (`docs/`)

All **design specs**, **implementation programs**, and **architecture notes** intended for the repository belong under **`docs/`** — not under `.github/design/` or other ignored trees.

## Layout

| Path | Purpose |
| ---- | ------- |
| **`design/`** | Frozen designs, brainstorming outputs, phased implementation plans (Markdown). |

Agents and contributors should add new design work under **`docs/design/`** using a **`YYYY-MM-DD-short-title.md`** (or **`implementation-plan`** suffix for executable checklists).

Product-facing docs remain at the repo root (**`README.md`**, **`SECURITY.md`**, **`QUICK_START.md`**) unless explicitly moved here by maintainers.
